# Contributing to LocAlign

Thank you for your interest in contributing to **LocAlign**.

Contributions are welcome in the form of bug reports, feature requests, documentation improvements, and code changes.

---

## How to contribute

1. Fork the repository and create a feature branch from `main`.
2. Make focused, well-documented changes.
3. Ensure the application runs locally and passes basic checks.
4. Commit changes with clear, descriptive messages.
5. Open a pull request describing the motivation and scope of your changes.

---

## Development guidelines

- Keep the Shiny app logic thin; place reusable logic in the `R/` directory.
- Do not commit large data files, databases, or generated results.
- Follow existing code style and structure.
- Add or update tests when modifying core functionality.
- Update documentation when behavior or interfaces change.

---

## Issues

Use GitHub issues to:
- Report bugs (include steps to reproduce when possible)
- Request new features
- Ask usage or installation questions

## Contact

If you are unsure whether a contribution is appropriate or how to proceed, feel free to open an issue to discuss it first.
